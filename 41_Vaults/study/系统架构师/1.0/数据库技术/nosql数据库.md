# 关系型数据库缺点
不满足高并发读写需求
不满足海量数据的高效率读写
不满足高扩展和可用性：很能进行横向扩展
# CAP理论
|  主要概念   |              解释               |
| :-----: | :---------------------------: |
|  C一致性   |   更新操作完成后所有节点在同一个时间的数据完全一致    |
|  A 可用性  |            服务一直可用             |
| P 分区容错性 | 分布式系统遇到某节点或网络分区故障时，仍然可以对外提供服务 |

CA 优先保证一致性和可用性，不再是分布式系统
CP 优先保证一致性和分区容错性，放弃可用性，牺牲用户体验
AP 优先保证可用性和分区容错性，全局数据可能不一致
# BASE理论
基本可用：允许损失部分可用性
软状态：状态有一段时间不同步
最终一致性：经过一段时间后达到一致性

# 主要类型
## 键值数据库
ky数据库
相关产品：redis等
优点：扩展性好，灵活性好，大量写操作时性能高
缺点：无法存储结构化数据，条件查询效率低
## 列族数据库
相关产品：HBase
典型应用：分布式存储和管理
优点：查找速度快，可扩展性强，容易进行分布式扩展
缺点：功能较少，大都不支持强事物一致性

## 图形数据库
相关产品：neo4j
数据结构：图结构
典型应用：专门用于处理具有高度相互关联关系的数据，比较适合于社交网络，模式识别，依赖分析
