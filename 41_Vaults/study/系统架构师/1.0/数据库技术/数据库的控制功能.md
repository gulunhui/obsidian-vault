# 事务管理
事务是数据库运行时的基本单位，事务相当于操作系统的进程
## 特性
1. 原子性。一个事务中的操作要么全部完成，要么全部都不做。如果出错就会rollback
2. 一致性。在事务开始之前和结束之后，数据的完整性没有被破坏
3. 隔离性。数据库允许多个并发事务对同一个数据进行读写和修改能力，隔离性可以防止多个事务并发执行时由于交叉执行而导致数据不一致
4. 持久性。事务结束后对数据的修改是永久的。

# 故障恢复
1. 事务故障的恢复。事务故障是指事务未运行至正常终止点前被撤销，这是恢复子系统应对此事务做撤销处理。
	1. 反向扫描日志文件。
	2. 对该事务的更新操作做逆操作
	3. 继续反向扫描日志文件，查找该事务的其他更新操作
	4. 如此继续，直至事务的开始标记
2. 系统故障。
	1. 未完成事务对数据库的更新已写入数据库
	2. 已提交的事务对数据库更新还留在缓存区
	3. 正向扫描文件
	4. 对撤销队列中的事务进行撤销处理
	5. 对重做队伍中各个事务进行重做处理。
3. 介质故障与病毒破坏的恢复。
	1. 装载最新备份
	2. 从故障点开始反向读文件
	3. 从起始点正向读取日志，根据重做标志，重做事务
