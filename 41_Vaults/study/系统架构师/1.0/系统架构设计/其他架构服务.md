# MVC

控制器 controller
	接受用户请求
	调用模型响应用户请求
	选择视图显示响应结果
视图 view
	显示模型状态 
	接受数据更新请求
	把用户输入数据传给控制器
模型
	代表应用程序查询状态
	响应状态查询
	处理业务流程
	通知视图业务状态更新
## 优点
1. 允许多种用户界面的扩展
2. 易于维护
3. 功能强大的用户界面
## 产生的问题
1. controller既处理业务，又处理拼接视图，不符合单一职责
2. 视图依赖过多
3. model既处理前端业务逻辑，又处理数据，不符合单一职责
# MVP
## 优点
1. 低耦合
2. 可以更高效使用模型
3. 复用性好
4. 可测试性好
