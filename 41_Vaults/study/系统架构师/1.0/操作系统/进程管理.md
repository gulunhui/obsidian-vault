# 基本概念
进程通常是由程序、数据集合、进程控制块PCB组成。PCB是一种数据结构
线程方式：把PCB放在一张线性表中
链接方式：把同一状态PCB，用链接字链接成一个队列
索引方式：将相同状态的PCB归入到一个索引表，多个状态对应多个索引表

进程三状态：
运行态：进程在处理机上运行时
就绪态：获取了处理机外一切资源，获取处理机就可以运行
阻塞态：进程等待某一事件发生而暂停停止运行

# 进程的通信
进程的互斥：多个进程因争用临界资源而互斥执行。
临界资源，一次只能被一个进程使用的资源

进程的同步：有一些进程是需要互相合作，协同工作的。

## PV操作
信号量S
P操作：将信号量S的值-1，即S=S-1
如果S>=0 ，进程继续执行，否则将进程置为阻塞

V操作：将信号S值+1
如果S>0，则继续执行。否则说明等待队列中有等待进程，需要唤醒等待进程

同步关系=信号量个数

1. 分配信号量：从左到右，从小打大
2. 填写pv操作：第一个进程没有p操作。最后一个进程没有V操作
