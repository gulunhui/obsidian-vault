# 软件开发方法分类
软件开发所遵循的办法和步骤
按照开发风范：
	自顶向下：将大问题分化小若干个可以解决的小问题
	自底向上：从具体的器件，逻辑部件或相似系统开始
按照性质：
	形式化：基于严密的、数学上的形式机制的计算机系统研究方法
	非形式化：各种开发模型
适用范围：
	整体性方法
	局部性方法

# 非形式化开发方法
## 结构化开发
结构化分析、结构化设计，结构化程序设计组成
1. 开发目标清晰化
2. 开发工作阶段化
3. 开发文档规范化
4. 设计方法结构化
缺点：开发周期长，难以适应需求变化，很少考虑数据结构

## 构件化开发
component
构件和对象的差别：构件并非一定包含类，一个类元素只能属于一个构件
构件
	独立部署单元
	作为第三方的组装单元
	没有状态
对象
* 一个实例单元，具有唯一标示
* 可能具有状态，此状态外部可见
* 封装了自己的状态和行为
### 构件的获取
1. 从现有的构件中获取符合要求的，可复用的
2. 通过遗留工程
3. 购买
4. 开发新构件
### 构件的分类
关键字分类：将应用领域的概念按照从抽象到具体的顺序逐次分解为树形有向无回路图结构
刻面分类法：若干用于刻画构件特征的刻面
超文本方法：任意跳转
### 构件的复用方法
检索与提取方法
理解与评价构件
修改构件
构件组装
### 构件检索方式
类似于构件分类方式
## 面向服务的方法 SO
服务总线ESB
抽象级别
	操作：最底层
	服务：代表操作的逻辑分组
	业务流程：最高层。为了实现特定业务目标而执行的一组长期运行的动作或活动。
## 原型法
快速原型法，根据用户初步需求利用系统工具快速建立一个系统模型，与用户交流
按照实现划分
	水平原型：行为原型，用于界面。细化需求但并未实现功能
	垂直原型：结构化原型，用于复杂算法的实现。实现了部分功能
按照最终结构划分
	抛弃式：探索式，需求不确定性，二义性，不完整性，含糊性
	演化式：逐步演化为最后系统，用于易于升级和优化的场合
## 敏捷开发
是一种以人为核心、迭代、循序渐进的开发方法
### 特征
1. 是适应性而非预设性。
2. 面向人的而非面向过程
### 核心思想
1. 适应性的
2. 以人为本
3. 迭代增量式的开发过程
### 主要内容
核心价值观：沟通、简单、反馈、勇气
最佳实践：计划游戏、小型发布、隐喻、简单设计、测试先行、重构、结对编程、集体代码所有制、持续集成、每周工作40个小时、现场客户、编码标准
### 方法分类
xp极限编程：高效、低风险、测试先行
cockburn水晶方法：不同项目、不同策略
SCRUM并列争求：迭代，30天为一个周期，按照需求优先级
FDD功用驱动方法：开发人员分类
开放式源码：虚拟团队
ASD自适应方法：预测——协作——学习
