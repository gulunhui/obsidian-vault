软件生存周期模型又称开发模型或软件过程模型。
模型的主要特点就是简单化、软件过程模型是软件开发实际过程的抽象和概括
软件过程模型概念：软件过程是制作软件产品的一组活动及结果。
主要活动有：
1. 软件描述
2. 软件开发
3. 软件有效性验证
4. 软件进化

# 瀑布模型
结构化方法的模型
特点：
	上次开发成果作为本次活动输入
	本次活动的成果输出到下次活动
缺点：
	需求难以确定
	需要很长时间才能得到软件初始版本
1. 计划
2. 需求分析
3. 设计
4. 编码
5. 测试
6. 运维
# 原型模型
原型法
用来确定用户需求
交流——快速计划——快速设计方法建模——构建原型
注意事项：
* 用户对系统模糊不清，无法准确回答目标系统的需求
* 要有一定的开发环境和工具支持
* 经过对原型的若干次修改，应收敛到目标范围内
* 对大型软件来说，可以不用太考虑
# 螺旋模型
大型复杂项目适用
原型+瀑布模型的结合
**有风险**
制定计划，风险分析，实施工程和客户评估
# 喷泉模型
面向对象方法，自底向上开发
没有明显界限
分析
设计
实现
维护
演化
# 基于可重用构件模型
构建化开发
软件需求
标识构件
检索构件库 开发构件，取出构件
集成目标系统
# RAD快速应用开发
强调极短的开发周期，是瀑布模型高速变种，通过使用基于构件的开发方法
规划
设计  实现
运行
基本思想：
1. 让用户主动参与到分析和设计中来
2. 让项目开发组织成一系列重点研讨会
3. 通过一种迭代的构造方法，快速需求分析和设计阶段
4. 让用户提前看到可工作的系统
# RUP统一过程模型
重型模型
x轴为：初始、细化、构建、交付
y轴：商业建模、需求、分析和设计、实现、测试、部署、配置和变更管理、项目管理、环境
RUP将业务建模，分析与设计、统一起来，贯穿整个开发过程。
分为四个阶段：初始阶段，细化阶段，构建阶段和移交阶段。每个阶段结束后还要进行项目评审
强调使用迭代和增量的方法来开发
好处：
1. 在开发早期就可以对关键的、影响大的风险进行处理
2. 可以提出一个软件体系结构来指导开发
3. 更好处理不可避免的需求变更
4. 较早得到一个可运行的系统
5. 为开发人员提供一个更有效的工作过程
