一种基于分布对象技术、强调通过可复用构件设计与构造软件系统的软件复用途径。
强调构件是购买而非重新构造
# 特征
1. 可组装
2. 可部署
3. 文档化
4. 独立化
5. 标准化

# 基于构件的软件工程步骤
1. 系统需求概览
2. 识别候选构件
3. 根据发现的构件修改需求
4. 体系定制与适配
5. 组装构件、创建系统
## 与传统的软件开发过程的不同
1. 早期完整的需求，以便尽可能多的识别出可复用的构件
2. 早期阶段根据可利用的构件来细化和修改需求
3. 架构设计完成后，可能需要修改构件以适合功能和构架的需求
4. 开发过程就是组装过程，有时候需要开发适配器
5. CBSE设计阶段特别重要，决定和限制了可选构件的范围
## 构件组装
顺序组装、层次组装、叠加组装
常见有参数不兼容，操作不兼容和操作不完备
## 缺点
1. 需要经验丰富的架构师
2. 可重用构件可能会牺牲系统性能
3. 使用构件组装应用程序时，增加研发人员的学习成本
4. 第三方构件库质量问题
