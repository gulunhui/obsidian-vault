# 软件重用
在两次或多次不同的软件开发过程中重复使用相同或相似软件元素的过程。
横向重用：不同领域中的软件元素
纵向重用：在一类具有较多公共性和应用领域之间进行软件重用
# 逆向工程
重构：在同一抽象级别上转化系统描述形式
设计恢复：借助工具从已有程序中抽象出有关数据设计、总体结构设计、过程设计等信息
再工程：在逆向工程所获得信息的基础上，修改或重构已有系统，产生系统一个新的版本。
正向工程：不仅从现有系统中恢复信息，使用该信息去改变或者重构系统，以改善质量

4个抽象层次
1. 实现级别：汇编语言
2. 结构级别：调用图、结构图、数据结构
3. 功能级别：反映程序段功能及程序段之间关系的信息
4. 领域级别：E-R模型
完备性：可以用在某一个抽象层次上提供信息的详细描述。抽象层次越高完备性越低
