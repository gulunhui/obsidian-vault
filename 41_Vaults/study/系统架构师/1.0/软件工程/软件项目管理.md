进度管理的过程：
1. 活动定义
2. 活动排序
3. 活动资源估算
4. 活动历时估算
5. 制定进度计划
6. 进度控制
# 范围管理
工作分解结构的原则：
1. WBS的工作包是可控和可管理的，不能过于复杂
2. 任务分解也不能过细，一般原则WBS的树形不能超过6层
3. 每个工作包要有一个交付成果
4. 每个任务必须有明确定义的完成标准
# 配置管理
核心内容
1. 版本控制：追踪文件变更
2. 变更控制：管理变更、保证变更有序进行
# 软件质量
就是软件于明确和隐晦的定义的需求相一致程度。
质量保证主要目标
1. 事前预防
2. 及时捕获缺陷
3. 作用于过程而非产品
4. 贯穿所有过程
# 风险管理
主要目的是预防风险
管理体系
1. Boehm体系：
	1. 风险估计：辨识、分析、排序
	2. 风险控制：管理计划、处理、监督
2. charette体系：
	1. 风险分析：辨识、估计、评价
	2. 风险管理：计划、控制、监督
3. CMU-SEI风险管理体系：全过程，与CMMI过程
## CMMI 软件能力成熟度模型
五个等级基本特征：
1. 初始级：过程随意混乱、成本超支
2. 一管理级：确保项目级的过程，建立明确目标
3. 已定义级：有标准流程，制度化、组织过程资产收集
4. 量化管理：定量目标、可预测
5. 优化级：最高级、创新优化、关注绩效
