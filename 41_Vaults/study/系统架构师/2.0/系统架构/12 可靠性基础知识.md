# 基本概念
可靠性是软件产品在规定条件下和规定时间区间完成规定功能的能力
## 不同点
1. 复杂性
2. 物理退化
3. 唯一性
4. 版本更新快
## 定量描述
### 可靠度
规定条件下、规定时间内不发生失效的概念
### 失效强度
单位时间软件系统出现失效的概念
### 平均失效前时间
MTTF 

### 平均恢复前时间
MTTR：从出现故障到修复成功的一段时间

### 平均故障间隔时间
MTBF：包括故障时间到维护设别时间
## 广义可靠性测试和侠义可靠性测试
为了最终评价软件系统的可靠性而运用建模、统计、试验、分析和评价等一系列手段对软件系统实施的一种测试
为了获取可靠性数据，按照预先确定的测试用例，在软件的预期使用环境中，对软件实施的一种测试
![[Pasted image 20251009220000.png]]

目的：
1. 发现软件系统的缺陷
2. 为软件的使用和维护提供可靠性依据
3. 确定软件是否达到可靠性定量
# 可靠性建模
## 影响可靠性因素
1. 运行环境
2. 软件规模
3. 软件的内部结构
4. 软件的开发方法和开发环境
5. 软件可靠性投入
## 可靠性模型的组成
1. 模型的假设
	1. 代表性假设
	2. 独立性假设
	3. 相同性假设
2. 性能度量
3. 参数估计方法
4. 数据要求
## 建模方法
1. 基于可靠的假设
2. 简单
3. 计算一些有用的亮
4. 给出未来失效行为的好的映射
5. 可广泛应用
