# MVC模式
主动模式
![[study/系统架构师/2.0/系统架构/_attachments/Pasted image 20250927173903.png]]

模型：执行业务流程，存储数据
视图：展示模型中的数据
控制器：把模型接受的事件和用户输入的数据转化对模型方法的调用。
优点：
1. 允许多种用户界面的扩展
2. 易于维护
3. 功能强大的用户界面

问题：
1. 控制层及处理逻辑，又生成视图
2. 视图依赖过多
3. 模型即处理前端逻辑又处理后端数据库访问细节
# MVP
优点：
1. 低耦合
2. 可以更加高效使用模型
3. 复用性好
4. 可测试性好
