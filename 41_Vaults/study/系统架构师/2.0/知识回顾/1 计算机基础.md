# 校验
CRC 循环冗余校验码
例如，数据信息为 1100，生成多项式为 X³+X+1（即 1011），则需先确定校验位长度为 3，将数据信息 1100 左移 3 位得到 1100000，再用 1011 对其进行模 2 除，得到余数 010，所以 CRC 编码是 1100010。

# 性能指标：
总线带宽：单位时间内总线可传输的最大数据量，反映总线的数据传输速率（吞吐量）。
- 带宽（B）= 总线宽度（W，单位 bit）× 总线频率（F，单位 Hz）/ 8（换算为字节），部分场景需考虑编码效率（如 PCIe 的 8b/10b 编码需 ×0.8）。
- 单位：MB/s（兆字节 / 秒）或 GB/s（吉字节 / 秒），例：32 位总线、133MHz 频率，带宽 = 32×133×10⁶ /8=532MB/s。
总线服务质量QoS
- 核心指标：带宽保障（最小带宽承诺）、时延上限（最大允许时延）、可靠性（无差错传输概率）、优先级调度支持。
- 实现机制：优先级仲裁（高优先级设备优先占用总线）、带宽预留（为关键应用分配固定带宽）、流量控制（避免总线拥堵）。

总线时延：从设备发起总线请求，到数据传输完成的总时间，反映总线的响应速度。
- 计算公式：总时延 = 仲裁时延 +（地址位数 + 数据位数）/ 总线带宽 + 握手周期 × 总线时钟周期。
总线抖动：同一类型传输的时延波动范围（即实际时延与平均时延的差值），反映总线传输的稳定性。

# 通信技术
香农公式：C = B × log₂(1 + S/N)
- C：信道容量，即信道的最大无差错传输速率（单位：bit/s，比特 / 秒）。
- B：信道带宽（单位：Hz，赫兹），指信道能传输的信号频率范围宽度。
- S/N：信道的信噪比（无量纲），即信号功率（S）与噪声功率（N）的比值，反映信道的噪声干扰程度。

# 计算机语言
UML 建模语言
## 静态图：
类图： 描述结构、属性、方法及类之间的关系
- 关联（Association）：对象间的静态连接（如 “学生 - 课程” 是多对多关联），可标注关联名、多重度（1:1、1:N、M:N）。
- 聚合（Aggregation）：整体与部分的 “松散关系”（部分可独立存在，如 “班级 - 学生”），用空心菱形 + 实线表示（菱形在整体端）。
- 组合（Composition）：整体与部分的 “紧密关系”（部分不可独立存在，如 “人 - 心脏”），用实心菱形 + 实线表示（菱形在整体端）。
- 继承（Generalization）：子类继承父类的属性和方法，用空心三角 + 实线表示（三角指向父类）。
- 实现（Realization）：类实现接口的方法，用空心三角 + 虚线表示（三角指向接口）。
- 依赖（Dependency）：一个类依赖另一个类的功能（临时性关联，如 “订单类依赖支付类”），用虚线 + 箭头表示（箭头指向被依赖方）。

对象图：
类图的实例化快照，描述某一时刻系统中对象的状态及对象间的关联。

接口图：
- 核心作用：描述接口的定义（仅含方法签名，无实现）及类与接口的实现关系。

包图：
- 核心作用：对系统中的类、接口、其他包进行分组，简化复杂系统的结构视图（如按功能模块划分 “用户包”“订单包”）。
- 关键元素：包（矩形带折叠角，标注包名）、包间依赖（虚线箭头）、包的嵌套（包内包含子包）。

部署图：
- 描述系统的硬件拓扑结构及软件构件在硬件上的部署关系
## 动态图
用例图
- 核心作用：描述系统的功能需求，展示参与者与系统用例之间的交互（从用户视角出发）。
- 关键元素：
    - 参与者（Actor）：系统的使用者或外部系统，用 stickman 表示（如 “用户”“管理员”）。
    - 用例（Use Case）：系统的一个功能模块，用椭圆表示（如 “登录”“下单”）。
    - 关系：关联（参与者与用例的交互，实线）、包含（用例 A 必须依赖用例 B，如 “下单” 包含 “支付”，用 “include” 标注）、扩展（用例 A 可选扩展用例 B，如 “登录” 扩展 “忘记密码”，用 extend 标注）、泛化（参与者 / 用例的继承，如 “管理员” 是 “用户” 的泛化）。

时序图
- 核心作用：按时间顺序描述对象间的交互过程（如 “用户登录” 的步骤：用户→系统提交账号密码→系统→数据库验证→系统→用户返回结果）。
- 关键元素：
    - 生命线（Lifeline）：对象的时间存在周期，用垂直虚线表示（顶端是对象名）。
    - 消息（Message）：对象间的交互指令，用实线箭头（同步消息）或虚线箭头（异步消息）表示，标注消息内容（如 “login (username, password)”）。
    - 激活期（Activation）：对象执行操作的时间区间，用生命线中的矩形表示。
    - 返回消息：用虚线箭头 + 空心三角表示（可省略，默认同步消息有返回）

协作图
- 核心作用：与时序图等价，侧重描述对象间的交互关系（空间结构），而非时间顺序（如 “登录” 涉及 “用户”“系统”“数据库” 三个对象的协作）。
- 与时序图的区别：时序图强调 “时间顺序”，协作图强调 “对象关联”，可相互转换

状态图
- 核心作用：描述单个对象的生命周期状态及状态间的转换（如 “订单” 的状态：待支付→已支付→待发货→已发货→已完成）。
- 关键元素：初始状态（实心圆点）、终止状态（实心圆点 + 圆圈）、状态（圆角矩形，标注状态名）、转换（箭头，标注触发事件，如 “支付成功”）。
### UML 建模原则

- 面向对象核心：封装、继承、多态（建模时需体现）。
- 适度抽象：根据建模目标选择合适的图（如需求阶段用用例图，设计阶段用类图、时序图）。
- 一致性：同一系统的不同图需保持逻辑一致（如类图中的类关系，在时序图中需对应）。
