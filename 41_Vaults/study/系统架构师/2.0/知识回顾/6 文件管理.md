## 位示图
记录文件存储器的使用情况。

用二进制位（bit）表示存储设备中每个物理块（或页框）的空闲状态，是一种紧凑的空间管理数据结构。
优势：占用空间小（如 1TB 硬盘，按 4KB 块划分，仅需 32KB 位示图）、查找高效（支持位运算快速定位空闲块）、实现简单。

- 关键参数：
    - 存储设备总物理块数 = 存储容量 / 物理块大小（需统一单位，如字节）。
    - 位示图总位数 = 总物理块数（1 位对应 1 块）。
    - 位示图存储单元：通常按字节（8 位）、字（如 32 位、64 位）组织，需明确 “位序号” 与 “存储单元序号 + 单元内位偏移” 的对应。
- 核心映射公式（假设位序号从 0 开始，从左到右、从上到下编号）：
    1. 物理块号 → 位示图位置：
        - 位序号 = 物理块号（1 位对应 1 块，直接映射）。
        - 存储单元序号（如字节号）= 位序号 // 每个单元的位数（如 8 位 / 字节则 ÷8，32 位 / 字则 ÷32）。
        - 单元内位偏移 = 位序号 % 每个单元的位数（如 8 位 / 字节则 %8，范围 0-7）。
    2. 位示图位置 → 物理块号：
        - 物理块号 = 存储单元序号 × 每个单元的位数 + 单元内位偏移。
