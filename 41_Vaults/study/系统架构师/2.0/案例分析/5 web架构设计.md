# 层次式架构
表现层：和用户交互
中间层：负责处理业务逻辑
数据访问层（持久层）：负责数据存储系统之间的交互

优点：
	开发人员只关心整个结构中的一层
	很容易用新的实现替换原有的层次实现
	降低层与层的依赖
	有利于标准化
	利于各层逻辑复用
	扩展性强
	安全性高
	项目结构清晰，分工明确
缺点：
	可能会导致性能问题
	建立清晰的分层架构比较难
# 表现层
## MVC模式
视图、控制器、模型
优点：允许多种用户界面扩展、易于维护、功能强大的用户界面

hibernate 将sql封装，使用简单。移植性更好
mybatis 半自动化，支持更复杂sql

## MVP模式
优点：
	模型视图完全分离
	可以更高效的使用模型
	可以将一个Presenter用于多个视图
	如果吧逻辑放到p中，可以脱离用户接口来测试这些逻辑

## MVVM模式
VVM中数据可以实现双向绑定
出现问题不易定位来源
