结构化分析步骤
1. 分析业务情况，做出放映当前物理模型的数据流图
2. 推导出等价的逻辑模型DFD
3. 设计新的逻辑系统，生成数据字典和基元描述
4. 建立人机接口，提供可供选择的目标系统模型DFD
5. 确定各种方案的成本和风险等级，并对方案进行分析
6. 选择一种方案
7. 建立完整需求规约（设计的依据）需求规格说明书
## 数据流图 DFD
数据流图建模过程
1. 明确目标，明确系统范围
2. 建立顶层数据流图
3. 构建第一层数据流图分解图
4. 开发dfd层次结构图
5. 检查确认dfd图
数据流图作用
	是需求分析的手段
	概括系统颞部逻辑过程
	是存档的文字材料

绘制步骤
1. 绘制输入输出
2. 画DFD内部
3. 为每一个数据流命名
4. 为加工命名

注意事项
矩形 外部实体
圆形 加工
数据流的操作不能越过加工

数据流图平衡
	父图和子图之间的平
	子图内部的平衡
## 数据字典
作用
	保证设计时不会遗漏
	互相参照，便于系统修改
	由描述内容检索名称
	一致性校验和完整性校验

## 基于UML的面向对象需求分析过程
用例图表示需求
利用包图及类图表示目标系统总体框架结构

时钟也可以作为参与者
### 用例关系
包含关系
	父指向子的关系，必须一起执行
扩展关系
	子指向父用例，不需要一起执行。子用例是父用例的一种额外可能性
泛化关系
	子用例共同的抽取。继承的反关系

### 类关系
关联关系：拥有的关系
聚合关系：整体和部分的关系，生命周期不一致
组合关系：整体和部分的关系，生命周期一致
依赖关系：B的变化可能会引起A的变化
泛化关系：父类与子类的关系，是继承的反关系
实现关系

# 面向对象设计
过程
1. 设计用例实现方案
	1. 提取边界类、实体类、控制类
	2. 构造交互图
	3. 根据交互图美化类图
2. 设计技术支撑方案
3. 设计用户界面
4. 精华设计模型
顺序图
是一种交互图，展现了一种交互，由一组对象或参与者以及他们之间可能发送的消息构成。强调消息的时间次序的交互图

消息类型
1. 同步消息
2. 异步消息
3. 返回消息
4. 自关联消息

通信图
协作图，强调收发消息的对象或参与者的组织结构

状态图
描述一个状态机，它由状态、转移、事件和活动组成
