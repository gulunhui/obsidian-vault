# 微服务架构
优势：
1. 通过分解巨大单体式应用为多个服务方法解决了复杂性问题
2. 让每个服务能够独立开发
3. 每个微服务独立的部署
4. 微服务使得每个服务独立扩展

缺点：
1. 并非所有系统都能转成微服务
2. 部署更加复杂
3. 性能
4. 数据一致性问题

# 架构评估
## 面向评估期的质量属性
性能：资源的需求、资源管理、资源仲裁
可用性：错误检测、错误恢复、错误避免
可靠性：容错、健壮性
安全性：抵抗攻击、检测攻击、从攻击中恢复
可修改性：局部化修改、防止连锁放映、推迟绑定时间（可维护性、可扩展性、结构重组、可以执行）

## 重要概念
功能性
可变性
互操作性
